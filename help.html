BSCMail Help

BSCMail is an application that allows the user to staff an event with volunteers and create an email with the pertinant information.

Basic Usage

To use BSCMail, follow the steps below.

  1. Use the management screens (Manage Shifts, Manage Managers, Manage Volunteers, and Manage Email) to create the appropriate shifts, managers, volunteers, and email template for the event. BSCMail remembers the changes you make, so for repeat events, you may need to do less management.
  2. Fill in the information for an event in the Create Event screen.
  3. Preview the information in the Create Email screen.

Screens

The different screens of BSCMail are divided into four categories:

  • the Main Window, from which all other screens are launched;
  • the manage screens, which help you manage shifts, managers, volunteers, and the email template;
  • the create screens, which help you create events and emails; and
  • the help screens, which display this help screen and give you information about BSCMail.

Main Window

When you start BSCMail, the main window comes up. From here, you can go to any other window by pressing the corresponding button.

Manage Shifts

The manage shifts screen lets you manage the volunteer shifts for the event. The manage shifts screen has the following controls.

  • The list control on the left displays the defined shifts. Click on a shift to see its details in the element panel.
  • The up and down buttons move the selected shift up or down in the list.
  • The sort button sorts the list of shifts alphanumerically.
  • The element panel on the right displays the data for the selected shift.
    • The description textbox displays the textual description of the shift.
    • The angel checkbox displays whether or not the shift only accepts angel volunteers. If the checkbox is checked, then the shift can only accept angel volunteers. If the checkbox is unchecked, then the shift may accept any volunteer.
  • The user may enter new information for a shift in the element panel and click the save button to save the new information for the currently selected shift.
  • The user may click the delete button to delete the currently selected shift.
  • The user may enter information for a shift in the element panel and click the add button to create a new shift. The new shift appears at the end of the list.

Manage Managers

The manage managers screen lets you manage the list of managers available for the event. The manage managers screen has the following controls.

  • The list control on the left displays the defined managers. Click on a manager to see its details in the element panel.
  • The up and down buttons move the selected manager up or down in the list.
  • The sort button sorts the list of managers alphanumerically.
  • The element panel on the right displays the data for the selected manager.
    • The name textbox displays the name of the manager.
    • The email textbox displays the email address of the manager.
    • The phone textbox displays the phone number of the manager.
  • The user may enter new information for a manager in the element panel and click the save button to save the new information for the currently selected manager.
  • The user may click the delete button to delete the currently selected manager.
  • The user may enter information for a manager in the element panel and click the add button to create a new manager. The new manager appears at the end of the list.

Manage Volunteers

The manage volunteers screen lets you manage the list of volunteers available for the event. The manage volunteers screen has the following controls.

  • The list control on the left displays the defined volunteers. Click on a volunteer to see its details in the element panel.
  • The up and down buttons move the selected volunteer up or down in the list.
  • The sort button sorts the list of volunteers alphanumerically.
  • The element panel on the right displays the data for the selected volunteer.
    • The name textbox displays the name of the volunteer.
    • The email textbox displays the email address of the volunteer.
    • The angel checkbox displays the angel status of the volunteer. That is, it is checked if the volunteer is an angel, and it is unchecked if the volunteer is not an angel. Some shifts only accepts angels as volunteers.
  • The user may enter new information for a volunteer in the element panel and click the save button to save the new information for the currently selected volunteer.
  • The user may click the delete button to delete the currently selected volunteer.
  • The user may enter information for a volunteer in the element panel and click the add button to create a new volunteer. The new volunteer appears at the end of the list.

Manage Email

The manage email screen displays the email template used by BSCMail. You may edit the email template and save any changes by clicking the save button.

You can insert certain macros into the template. Macros are enclosed by angle brackets and are replaced in the final email by information from the event. The list of defined macros follows.

<email-addresses>
A comma-separated list of the email addresses of the event's manager and volunteers.
<subject>
A formatted subject line for the email. The subject line is in the format: "BSC Volunteer Schedule for Friday January 2".
<manager-name-allcaps>
The name of the event's manager, formatted in all caps.
<manager-first>
The first name of the event's manager.
<manager-phone>
The phone number of the event's manager.
<band>
The band for the event.
<instructors>
The instructors for the event.
<general-price>
The general admission price for the event.
<student-price>
The student/senior admission price for the event.
<discount-price>
The discount admission price for the event.
<shifts-sequential>
A list of all the shifts defined for the event, along with the names of the volunteers assigned to them, listed sequentially.
<shifts-compact>
A list of all the shifts defined for the event, along with the names of the volunteers assigned to them. Consecutive shifts with the same name are combined into a single line.
<long-date>
The date of the event, formatted like "Friday January 2".
